My First Post: My OS

Development Deatils of the OS:

When creating my 64 bit OS, I consulted the two recommended Youtube videos as a guide suggested by our instructor. I installed Visual Studio code as my text editor and Docker in order to build the image. I used docker for the building process and Qemu for the emulating process. I used Docker to create a reproducible build environment for building my operating system. I created a dockerfile which would contain and describe the steps needed to create the build environment image. My image was based on the pre-made image that which holds the gcc compilation tools I would need. I installed a few more packages like nasm so that my assembly code would compile and grub package in order to compile the final iso file. In order to have access to all my files (and is demonstrated in the tutorial), I connected a volume to the development environment (VOLUME /root/env).

After setup:

Once docker was running, I opened the terminal and entered the code to build my image where buildenv was the name of the folder my dockerfile resided in and myos-buildenv is the tag name I gave it:

docker build buildenv -t myos-buildenv

While executing this, I faced an error which I was unfamiliar with and upon researching discovered a line of code which would help me run it. Its purpose remains unclear but it helped me run my file:

DOCKER_BUILDKIT=0
